### ARCH-F-09: Test-file `#[path = "..."]` directive (P3)

**Audit finding:** `#[path = "..."]` directives were used to reference test
files from `src/`, which is an anti-pattern (test code leaked into source
tree). Cross-references NAME-F-04.

**Resolution path:** Resolved by **Phase 26 (v20.0 Module Tree Restoration)**,
which addressed the underlying NAME-F-04 ("Move qwen3/model_tests.rs,
qwen3_5/model_tests.rs, qwen3_5/speculative_tests.rs to tests/ or convert
to mod tests") by moving the test files out of `src/`. Once test files
live in `crates/<crate>/tests/` per Rust convention, `#[path]` directives
become unnecessary and are removed.

**Verification:**
```bash
grep -rn '#\[path' crates/ --include="*.rs"
# (empty — no anti-pattern #[path] directives remain)
```

### API-F-29: `DraftLoader::load` returns `Box<dyn ModelBackend>` — verify intent (P3)

**Audit finding:** The `DraftLoader::load` method returns
`Result<Box<dyn ModelBackend>, DraftRegistryError>`. The audit asked to
"verify intent" — i.e., confirm whether the boxed dynamic return is
intentional or a workaround for object-safety issues.

**Resolution path:** Resolved by **Phase 25 (v20.0 P0 Critical Fixes)**,
which addressed API-F-02 (P0 object-safety violations). That work made
`ModelBackend` properly object-safe (no generic methods, no `Self: Sized`
bounds). Once the trait satisfies object-safety, returning
`Box<dyn ModelBackend>` is the **canonical Rust dynamic dispatch pattern**
— not a workaround. The audit's "verify intent" question is answered:
*yes, this is intentional and now type-safe.*

**Verification:** `pub trait ModelBackend: Send + Sync { ... }` is
object-safe (dyn-compatible); the `dyn_safety.rs` compile-only test in
`crates/testing/tests/` exercises `Box<dyn ModelBackend>` usage across the
codebase.
