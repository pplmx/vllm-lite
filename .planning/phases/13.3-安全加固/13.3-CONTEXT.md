# Phase 13.3: 安全加固 - Context

**Gathered:** 2026-04-27
**Status:** Ready for planning
**Mode:** Autonomous

## Phase Boundary

Security hardening — TLS termination, mTLS for inter-node, API authentication, RBAC, audit logging, and structured logging with correlation IDs.

## Requirements

- SEC-01: TLS termination support via cert-manager integration
- SEC-02: mTLS for inter-node communication
- SEC-03: API authentication (JWT/API key validation)
- SEC-04: RBAC for multi-tenant access control
- SEC-05: Audit logging for API requests
- OPS-05: Structured logging with request correlation IDs

## Success Criteria

1. External HTTPS endpoint with valid certificate
2. Node-to-node communication encrypted (mTLS)
3. Unauthenticated requests rejected
4. RBAC restricts model access by role
5. Audit log captures all API calls with user identity
6. Structured logs include request_id correlation

## Codebase Context

Existing infrastructure:
- `crates/server/src/auth.rs` — Already has authentication middleware
- `crates/server/src/backpressure.rs` — Rate limiting
- `crates/server/src/main.rs` — Server setup

New components needed:
- TLS termination with rustls
- mTLS configuration for gRPC
- JWT validation
- RBAC middleware
- Audit logger
- Request correlation ID middleware
