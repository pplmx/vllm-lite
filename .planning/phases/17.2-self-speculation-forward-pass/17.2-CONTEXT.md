# Phase 17.2: Self-Speculation Forward Pass - Context

**Gathered:** 2026-05-13
**Status:** Ready for planning

<domain>

## Phase Boundary

SelfSpeculativeModel generates actual draft tokens via a layer-truncated forward pass through only 1/8 of the target model's layers, with weight sharing (zero-copy references to target weights) and a separate isolated KV cache. This replaces the current stub implementation that returns empty drafts.

Key deliverables:

- `SelfSpeculativeModel.generate_draft()` runs a real forward pass through `draft_layer_count` layers
- Draft generation uses greedy argmax sampling to select the most probable next token
- Draft and target maintain separate KV cache block IDs — no silent state corruption between passes
- Weight sharing uses zero-copy references to target model weights (no additional GPU memory allocated)
</domain>

<decisions>

## Implementation Decisions

### Layer-Truncated Forward Pass Design

- Add `forward_to_layer()` to `ModelBackend` trait with `upto_layer: usize` parameter — each model impl iterates layers up to the limit, applies norm + lm_head, returns logits
- Clone target layers' `Vec<Box<dyn TransformerBlock>>` at SelfSpeculativeModel construction — candle Tensors are Arc-based, clone is O(1) zero-copy

### Draft KV Cache Isolation

- Draft KV cache has `draft_layer_count` layers with same head_dim/num_heads as target — only cache what the draft layers need
- Create `PagedKvCache` in SelfSpeculativeModel with its own block pool, separate from scheduler-managed target blocks
- Reuse MemoryManager::rollback() from Phase 17.1 for the draft model's KV cache

### Draft Generation Loop

- Use greedy argmax (temperature=0) for draft sampling — simplest, deterministic, matches config's sample_token()
- Engine's batched draft generation calls SelfSpeculativeModel.generate_draft() which runs truncated forward per position
- Run truncated forward sequentially per sequence — draft batch is per-sequence (not per-position like main decode)

### the agent's Discretion

Implementation details of the `forward_to_layer()` trait signature, exact PagedKvCache creation for draft, and the batch-level draft generation wiring are left to the agent's discretion.

</decisions>

- `TransformerBlock` trait — `crates/model/src/components/block.rs` (per-layer forward)
- Sample concrete model: `Qwen3Model` at `crates/model/src/qwen3/model.rs` with layer loop at lines 357-378

### Established Patterns

- All models iterate `self.layers.iter_mut().enumerate()` in forward — can be truncated with `.take(upto_layer)`
- KV cache is built with `PagedKvCache::new(num_layers, num_heads, head_dim, num_kv_blocks, device, false)`
- `Engine.speculative.rs` calls `draft_model.lock().unwrap().forward()` — wrapped as `Arc<Mutex<dyn ModelBackend>>`
- Phase 17.1 established `MemoryManager::rollback()` and batched draft generation

### Integration Points

- `SelfSpeculativeModel.generate_draft()` — needs real truncated forward implementation
- `ModelBackend` trait — needs `forward_to_layer()` method added
- Concrete model impls (Qwen3, Llama, etc.) — need `forward_to_layer()` per model
- `Engine` speculative path — uses `draft_model.forward()` currently, should use `SelfSpeculativeModel`'s DraftVerifier trait instead
- Architecture registry — if building draft from individual blocks, `Architecture::create_block()` can create per-layer blocks

## Specific Ideas

- `forward_to_layer()` signature: `fn forward_to_layer(&mut self, seq_ids: &[SeqId], input_tokens: &[Vec<TokenId>], positions: &[Vec<usize>], kv_block_ids: &[Vec<usize>], num_computed_tokens: &[usize], is_prefill: &[bool], upto_layer: usize) -> Result<BatchOutput>`
- The draft model needs its own PagedKvCache with draft_layer_count layers, allocated during SelfSpeculativeModel construction
- Weight sharing: clone target model's embed_tokens, layers[0..draft_layer_count], norm, lm_head tensors — all O(1)
- The `#![allow(dead_code)]` at the top of self_spec.rs should be removed when implementation is real
