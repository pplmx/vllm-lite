<decisions>
## Implementation Decisions

### Adaptive Depth Enhancement (ADPT-01, ADPT-02, ADPT-03)
- Add EWMA with configurable alpha (default 0.1) to DraftAccuracyTracker — replaces simple sliding-window average for smoother rate tracking
- Implement deadband hysteresis in maybe_adjust() — only adjust when rate deviates beyond target ± threshold (default 0.05), reset cooldown on no-change
- Keep current cooldown_steps counter — prevents oscillation by requiring N steps between adjustments

### Benchmark Suite (BENCH-01, BENCH-02, BENCH-03, BENCH-04)
- New benches/speculative.rs in the workspace benches crate — follows existing benchmark convention
- Report P50/P95/P99 latency + tokens/sec throughput + speculative efficiency + acceptance rate
- Workload types: single-sequence, multi-sequence (4 concurrent), multi-sequence (16 concurrent) — with and without warmup
- Comparison baseline: non-speculative vs speculative (fixed depth) vs adaptive speculative — all three in A/B output table

### the agent's Discretion
Exact EWMA alpha value selection, deadband threshold constants, benchmark warmup iteration count, and benchmark output formatting are left to the agent's discretion.

<specifics>
## Specific Ideas

- EWMA formula: `smoothed_rate = alpha * current_rate + (1 - alpha) * previous_smoothed_rate`
- Start EWMA with first observed rate (not 0) to avoid initialization bias
- Deadband: `|rate - target| <= threshold` → no adjustment, else adjust in direction of deviation
- Benchmark should use proper criterion Criterion::default().sample_size() for statistical rigor
- Warmup: at least 10 iterations before measurement begins per criterion convention

</specifics>
