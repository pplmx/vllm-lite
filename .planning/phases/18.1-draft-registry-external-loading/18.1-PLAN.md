# Phase 18.1 Plan: Draft Registry + External Loading

## Requirements

- MMLT-01: Engine can load a separate draft model instance (different architecture and/or size from target)
- MMLT-02: External draft uses its own `ModelBackend` instance with independent KV cache block IDs (no state leak with target)
- MMLT-03: Draft weights loaded lazily — deferred until first request selects this draft model
- LIFE-01: `DraftModelRegistry` provides register / load / unload operations for draft models at runtime

## Implementation

### 1. Add core types in `crates/core/src/speculative/draft_registry.rs`

**Public types:**
```rust
pub struct DraftId(pub String);

pub struct DraftSpec {
    pub id: DraftId,
    pub model_dir: std::path::PathBuf,
    pub arch_hint: Option<String>,
    pub kv_blocks: usize,
    pub ref_count: usize,  // starts at 0; updated by routing in 18.3
}

pub struct LoadedDraft {
    pub spec: DraftSpec,
    pub backend: Box<dyn vllm_traits::ModelBackend>,
    pub block_allocator: BlockAllocator,  // isolated block pool
}

pub enum DraftState {
    Unloaded(DraftSpec),
    Loaded(LoadedDraft),
}

pub struct DraftModelRegistry {
    drafts: RwLock<HashMap<DraftId, DraftState>>,
    device: Device,
}

#[derive(Debug, thiserror::Error)]
pub enum DraftRegistryError {
    #[error("unknown draft id: {0}")]
    UnknownDraftId(DraftId),
    #[error("draft already loaded: {0}")]
    AlreadyLoaded(DraftId),
    #[error("draft still in use (refcount={0})")]
    InUse(usize),
    #[error("draft load failed: {0}")]
    LoadFailed(String),
}
```

**Public methods on `DraftModelRegistry`:**
- `new(device: Device) -> Self`
- `register(&self, spec: DraftSpec) -> Result<(), DraftRegistryError>` — inserts as `Unloaded`, fails if id already exists
- `preload(&self, id: &DraftId) -> Result<(), DraftRegistryError>` — eagerly load (optional path)
- `get_or_load(&self, id: &DraftId) -> Result<(), DraftRegistryError>` — lazy load on first call
- `unload(&self, id: &DraftId) -> Result<(), DraftRegistryError>` — transitions Loaded → Unloaded, frees blocks
- `lookup(&self, id: &DraftId) -> Option<DraftState>` — read-locked read; doesn't load
- `get(&self, id: &DraftId) -> Result<LoadedSnapshot, DraftRegistryError>` — returns cheap ref to loaded state (read lock) — caller must release
- `increment_ref(&self, id: &DraftId)` and `decrement_ref(&self, id: &DraftId)` — refcount ops for 18.3
- `ids(&self) -> Vec<DraftId>` — list all registered ids
- `len(&self) -> usize` — count

The `get` method returns a `DraftSnapshot` guard that holds the read lock until dropped, so callers can iterate multiple drafts without lock churn.

**Loading implementation:**
- Use existing `ModelLoaderBuilder` from `vllm_model::loader`
- Construct `ModelLoader::builder(device).with_model_dir(spec.model_dir.to_string_lossy().into_owned()).with_kv_blocks(0).build()` (kv_blocks on loader is unused for drafts; the draft owns its own allocator)
- Call `loader.load()` → `Box<dyn ModelBackend>`
- Create a fresh `BlockAllocator::new(spec.kv_blocks)` and store in `LoadedDraft`

**Block isolation:**
- `LoadedDraft::block_allocator` is private to the draft — no sharing with target's `MemoryManager`
- `LoadedDraft::backend.forward(...)` will write to its own block ids only

### 5. Tests in `crates/core/src/speculative/draft_registry.rs`

```rust
#[cfg(test)]
mod tests {
    use super::*;

    fn dummy_spec(id: &str, kv_blocks: usize) -> DraftSpec {
        DraftSpec {
            id: DraftId(id.to_string()),
            model_dir: PathBuf::from("/nonexistent"),
            arch_hint: None,
            kv_blocks,
            ref_count: 0,
        }
    }

    #[test]
    fn test_register_creates_unloaded_state() {
        let registry = DraftModelRegistry::new(Device::Cpu);
        registry.register(dummy_spec("a", 64)).unwrap();
        assert!(matches!(registry.lookup(&DraftId("a".into())), Some(DraftState::Unloaded(_))));
    }

    #[test]
    fn test_register_duplicate_id_errors() {
        let registry = DraftModelRegistry::new(Device::Cpu);
        registry.register(dummy_spec("a", 64)).unwrap();
        assert!(registry.register(dummy_spec("a", 64)).is_err());
    }

    #[test]
    fn test_unload_unknown_id_errors() {
        let registry = DraftModelRegistry::new(Device::Cpu);
        assert!(registry.unload(&DraftId("nope".into())).is_err());
    }

    #[test]
    fn test_get_or_load_load_failure_keeps_unloaded() {
        let registry = DraftModelRegistry::new(Device::Cpu);
        registry.register(dummy_spec("bad", 64)).unwrap();
        let result = registry.get_or_load(&DraftId("bad".into()));
        assert!(result.is_err());
        // Still unloaded
        assert!(matches!(registry.lookup(&DraftId("bad".into())), Some(DraftState::Unloaded(_))));
    }

    #[test]
    fn test_refcount_increment_decrement() {
        let registry = DraftModelRegistry::new(Device::Cpu);
        registry.register(dummy_spec("a", 64)).unwrap();
        registry.increment_ref(&DraftId("a".into())).unwrap();
        registry.increment_ref(&DraftId("a".into())).unwrap();
        registry.decrement_ref(&DraftId("a".into())).unwrap();
        // Spec is still tracked; unload still errors because we don't auto-unload here (that's 18.2)
        // But manual unload should work since refcount is now 1 (not 0)
        // In 18.1, refcount is informational; unload is unconditional (Loaded → Unloaded)
    }

    #[test]
    fn test_ids_lists_all_registered() {
        let registry = DraftModelRegistry::new(Device::Cpu);
        registry.register(dummy_spec("a", 64)).unwrap();
        registry.register(dummy_spec("b", 32)).unwrap();
        let mut ids = registry.ids();
        ids.sort();
        assert_eq!(ids, vec![DraftId("a".into()), DraftId("b".into())]);
    }
}
```

### 6. Engine integration test in `crates/core/src/engine.rs` (existing tests section)

```rust
#[test]
fn test_engine_with_empty_drafts() {
    // Existing default — should still work
    let engine = Engine::default();
    assert_eq!(engine.draft_registry().len(), 0);
}

#[test]
fn test_engine_with_declared_drafts() {
    // Build a config with two draft specs (model_dir may not exist on disk)
    let mut config = EngineConfig::default();
    config.drafts = vec![dummy_spec("a", 64), dummy_spec("b", 32)];
    // Engine::new(config) should construct and register, but not load
    let engine = Engine::new(config).unwrap();
    assert_eq!(engine.draft_registry().len(), 2);
    assert!(matches!(
        engine.draft_registry().lookup(&DraftId("a".into())),
        Some(DraftState::Unloaded(_))
    ));
}
```

## Success Criteria

1. ✓ `DraftModelRegistry::register` stores a spec as `Unloaded` without touching disk
2. ✓ `DraftModelRegistry::get_or_load` triggers `ModelLoader` load on first call; subsequent calls hit cache
3. ✓ `LoadedDraft` owns a private `BlockAllocator` — block ids never collide with target's
4. ✓ `Engine::new(config)` with `config.drafts` populated creates registry and pre-registers all specs
5. ✓ Heterogeneous coexistence is supported: arch_hint is optional; loader auto-detects from config.json
6. ✓ `unload` is a no-op (with ok) on already-unloaded draft; fails on unknown id
7. ✓ All unit tests pass; `cargo clippy -D warnings` clean; `cargo fmt --check` clean
