# Phase 16.1: Architecture - Context

**Gathered:** 2026-04-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Define core speculation infrastructure: DraftVerifier trait, SpeculativeModel wrapper, SpeculationConfig, and RejectionStrategy enum. This phase establishes the trait/struct foundation that subsequent phases build upon.

</domain>

<decisions>
## Implementation Decisions

### Existing Infrastructure
The codebase already has partial speculative decoding support:
- `AdaptiveDraftConfig` in `crates/core/src/types.rs`
- `DraftAccuracyTracker` and `AdaptiveSpeculativeDecoder` in `crates/core/src/speculative/`
- `Engine::enable_adaptive_speculative()` method in `crates/core/src/engine.rs`

This phase extends the existing infrastructure, not creates from scratch.

### Code Location
All new types go in `crates/core/src/speculative/`:
- `verifier.rs` - DraftVerifier trait
- `model.rs` - SpeculativeModel wrapper
- `config.rs` - SpeculationConfig
- `strategy.rs` - RejectionStrategy enum
- Update `mod.rs` to export new modules

### the agent's Discretion
All implementation choices are at the agent's discretion — pure infrastructure phase. Follow existing Rust/candle patterns in the codebase.

</decisions>
