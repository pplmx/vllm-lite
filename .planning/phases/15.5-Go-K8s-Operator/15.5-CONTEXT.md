## Implementation Notes

### Controller Architecture
- Uses controller-runtime for standard K8s patterns
- VLLMEngine CRD with model, replicas, GPU config
- Deployment and Service owned resources
- Status tracking with conditions

### CRD Fields
- `spec.model`: Model name to serve
- `spec.replicas`: Number of replicas (default: 1)
- `spec.image`: Container image (default: ghcr.io/vllm/vllm:latest)
- `spec.gpuResource.count`: Number of GPUs
- `spec.tensorParallelSize`: TP degree

### Reconciliation Logic
- Create/update Deployment with vLLM server
- Create/update Service for HTTP endpoint
- Update status with service URL
- Handle scale up/down via replicas changes
