# Phase 18.2 Plan: Lifecycle + Memory Budget

## Requirements

- LIFE-02: Unloading a draft model frees its KV cache blocks via MemoryManager (no orphan blocks)
- LIFE-03: Registry tracks active drafts with reference counts; auto-unload when refcount hits zero
- MEM-01: Engine enforces total VRAM budget = target weights + target KV cache + N concurrent drafts
- MEM-02: Load-time weight-size estimation (from model loader metadata); runtime KV cache growth tracking
- MEM-03: Engine refuses to load a draft if estimated VRAM would exceed budget, with structured error

### 4. Extend `DraftModelRegistry` with budget-aware lifecycle

**`crates/core/src/speculative/draft_registry.rs`:**
- Add `budget: Arc<MemoryBudget>` field, set via `with_budget(budget)`
- New `attach_loaded_budgeted(id, backend) -> Result<(), DraftRegistryError>`:
  1. Lookup entry; if missing → `UnknownDraftId`
  2. Compute `estimated = spec.weight_size_estimate_bytes + spec.kv_blocks * BLOCK_BYTES`
  3. Call `budget.try_reserve_draft(estimated, Some(id.clone()))`; on failure → `MemoryBudgetExceeded`
  4. Call existing `attach_loaded` logic; on failure, release the reservation
  5. Update `budget.record_draft_kv_growth(estimated as i64)` to mark draft as "used"
- Modify `unload(id)`:
  1. If refcount > 0 → return `InUse(refcount)` error
  2. If `Loaded` → release budget (`budget.release_draft(estimated)`), drop backend, transition to `Unloaded`
  3. If `Unloaded` → return `Ok(())`
- Add `force_unload(id)` — same as `unload` but ignores refcount
- Modify `decrement_ref(id)`:
  1. Decrement (with floor at zero)
  2. If new count is zero AND state is `Loaded` → auto-unload (release budget, drop backend, transition to `Unloaded`)
  3. Return whether auto-unload occurred (for caller logging)
