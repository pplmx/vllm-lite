## Design Notes

1. **Atomic budget reservation** — `attach_loaded_budgeted` reads the spec
   under a read lock, attempts the budget reservation (which may fail), and
   only then takes the write lock to do the state transition. If the
   reservation fails, state is untouched. If a concurrent caller won the race,
   the reservation is rolled back via `release_draft`.

2. **Unload is auto-cleanup** — `unload` of a Loaded draft:
   - Checks refcount > 0 → `InUse`
   - Releases budget reservation (`budget.release_draft(estimated_total)`)
   - Drops the `LoadedDraft` (which drops the `Box<dyn ModelBackend>` and the
     `BlockAllocator`)
   - Transitions state to `Unloaded` with the cloned spec

3. **Auto-unload is silent (with side effect)** — `decrement_ref` returning
   `bool=true` lets the caller log/observe but doesn't fail. The auto-unload
   is the implementation of LIFE-03.

4. **Unlimited default** — `Engine::new` constructs a registry with
   `MemoryBudget::unlimited()` (total = u64::MAX). All v17 callers and
   existing tests continue to work without any code change. Budget enforcement
   is opt-in via `Engine::with_budget_boxed` or by calling
   `Engine::attach_draft_budgeted` on the default engine (which is also a
   no-op given unlimited budget).

5. **No preempt-to-make-room** — `attach_loaded_budgeted` either fits or
   fails. We do not evict loaded drafts to make room for a new one; that
   would violate their refcount semantics. Callers must `force_unload_draft`
   explicitly to free budget.
