## Implementation Decisions

### Helm vs Kustomize
Helm is more widely adopted for LLM serving (vllm, Ray Serve use Helm). Use Helm.

### Operator Language
Go is industry standard for K8s operators. Create separate `k8s/operator/` directory.

### Health Endpoint Paths
Follow K8s conventions: `/health/live`, `/health/ready`

### gRPC Framework
Use tonic + prost for Rust-native async gRPC.

### Node Discovery
Use Kubernetes headless service DNS for peer discovery.

## Success Criteria

1. `helm install vllm-lite ./charts/vllm-lite` deploys 3 replicas
2. `kubectl get pods` shows 3 running pods
3. `/health/live` returns 200 when process alive
4. `/health/ready` returns 200 when model loaded
5. Headless service returns peer pod IPs
6. gRPC ping/pong between pods succeeds
7. Rolling update completes without downtime
