## Implementation Details

### Commands
- `vllm config validate --file <path>` - Validate YAML config file syntax and schema
- `vllm model list --dir <path>` - List available models with names and sizes
- `vllm model info --path <path>` - Show model metadata (architecture, params)

### Config Validation (CLI-01)
- Parse YAML with serde_yaml
- Check required fields: server, engine
- Validate port range (1-65535)
- Validate kv_blocks > 0

### Model Listing (CLI-02)
- Scan directory for subdirectories
- Calculate total size per model
- Display name, size, detected architecture

### Model Info (CLI-03)
- Parse config.json for metadata
- Show: architecture, hidden_size, num_layers, num_heads, vocab_size
- Show: total_size, num_files

## Usage Examples

```bash
# Validate a config file
cargo run -p vllm-server --bin vllm -- config validate config.yaml

# List models in directory
cargo run -p vllm-server --bin vllm -- model list /models

# Show model info
cargo run -p vllm-server --bin vllm -- model info /models/llama-7b
```
