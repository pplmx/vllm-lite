# Phase 14.1: 基准测试 - Context

**Gathered:** 2026-04-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement standardized benchmarks for measuring throughput (tokens/sec) under concurrent load and latency (TTFT, P50, P95, P99). Use SchedulerObserver pattern for non-invasive metrics collection.

</domain>

<decisions>
## Implementation Decisions

### Benchmark Architecture
- Use existing BenchmarkSuite from benches/lib.rs
- Add ThroughputBenchmark and LatencyBenchmark structs
- Implement SchedulerObserver for metrics collection

### Warmup Strategy
- Discard first 3 iterations before measurement
- Use BenchmarkConfig::warmup_iterations (already supported)
- Add --warmup flag to CLI

### Metrics Collection
- TTFT: Time to First Token from request start to first decoded token
- P50/P95/P99: Percentiles from latency histogram
- Throughput: Total tokens / total time under concurrent load

</decisions>
