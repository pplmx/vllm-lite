## Fixed Issues

### WR-01: Test name claims sequence-degradation behavior that is not exercised

**Files modified:** `crates/core/tests/multi_draft_integration.rs`
**Commit:** `14e3419`
**Applied fix:** Renamed `test_fallback_runtime_error_marks_sequence_degraded` to
`test_stub_backend_fail_next_n` so the name matches what the test actually verifies
(StubBackend's `fail_next_n` flag returns Err on first call, Ok on second). Did not
add a new engine-driven FALL-02 test — that requires engine surface work not in scope
for a fix pass.

### WR-02: Misleading "concurrent" naming — tests are strictly sequential

**Files modified:** `crates/core/tests/multi_draft_integration.rs`
**Commit:** `d62bfbe`
**Applied fix:** Renamed both `test_budget_concurrent_n_drafts_within_budget` →
`test_budget_accepts_n_drafts_within_budget` and `test_budget_concurrent_n_plus_one_over_budget`
→ `test_budget_refuses_n_plus_one_drafts_over_budget`. Dropped the "concurrent" suffix
since neither test spawns threads.

### WR-03: `test_budget_refuses_n_plus_one_drafts_over_budget` does not verify state consistency after fallback

**Files modified:** `crates/core/tests/multi_draft_integration.rs`
**Commit:** `7dd24ac`
**Applied fix:** Added `assert!(!h.registry.is_loaded(&DraftId("b".into())));` immediately
after the `matches!(r, ResolvedDraft::SelfSpec(_))` assertion, locking in the post-fallback
invariant that draft `b` stays Unloaded.

### WR-04: `test_lifecycle_register_lazy_load_route_unload` does not verify the loader was invoked

**Files modified:** `crates/core/tests/multi_draft_integration.rs`
**Commit:** `b51126a`
**Applied fix:** Added `pub load_count: AtomicU64` to `MapLoader` (initialized to 0 in
`new()`, incremented in `DraftLoader::load`). The lifecycle test now asserts both
`load_count == 0` before `resolve` and `load_count == 1` after `resolve`.

### WR-05: `test_unload_with_nonzero_refcount_returns_inuse` does not verify state is preserved

`unwrap_err()` line, matching the equivalent assertion in `draft_registry.rs:758-771`.

### WR-06: `test_budget_accepts_n_drafts_within_budget` uses weak assertion

**Files modified:** `crates/core/tests/multi_draft_integration.rs`

### WR-07: Bench measurement semantics inconsistent with group name

**Files modified:** `crates/core/benches/multi_draft_speculative.rs`
**Commit:** `ced350c`
