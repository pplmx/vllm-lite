<action>
Spawn naming audit subagent with concrete methodology:

**NAME-01 File naming:**
- `find crates -name "*.rs" -o -name "*.toml" | xargs -I {} basename {} | grep -E "^[0-9]+_"` → stage-info named files
- Check for files with hyphens (Rust prefers underscores)
- Check for non-snake_case filenames
- Cross-check filename matches module declaration

**NAME-02 Type naming:**
- `grep -rh "^pub struct\|^pub enum\|^pub trait" crates/ | sort -u`
- Check PascalCase: `grep -E "^pub (struct|enum|trait) [a-z]+"`
- Flag redundant suffixes: `grep -E "Info|Data|Manager|Helper|Util$"` (when semantic already clear)
- Flag abbreviations that hurt readability

**NAME-03 Function/method naming:**
- `grep -rh "^pub fn\|^pub async fn\|^    pub fn\|^    pub async fn" crates/ | sort -u`
- Check snake_case: `grep -E "fn [A-Z]"`
- Check verb consistency (get/set/with/is/has)
- Flag inconsistent prefix usage (e.g., `fetch_*` vs `get_*` mixing)

**NAME-04 Variable naming:**
- For top-20 largest files, scan for single-letter variable names (non-loop)
- `grep -rE "\b(let|let mut) [a-z]\b" crates/`
- Flag descriptive-but-confusing names (e.g., `data` without type context)
- Find same-name-different-meaning or different-name-same-meaning pairs

**NAME-05 Module naming:**
- For each `.rs` file, extract `mod foo;` declarations and check match
- `grep -rh "^pub mod\|^    pub mod\|^mod " crates/`
- Check nesting depth consistency (max depth varies per crate)

Output REPORT.md (detailed tables per dimension) and SUMMARY.md (P0/P1/P2 prioritized table with file:line citations).
