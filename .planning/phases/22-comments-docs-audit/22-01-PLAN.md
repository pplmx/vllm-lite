<action>
Spawn documentation audit subagent with concrete methodology:

**DOCS-01 Doc-comment coverage:**
- For each crate, count `pub` items: `grep -rE "^pub (fn|struct|enum|trait|mod|use)" crates/$CRATE/src | wc -l`
- For each crate, count `pub` items with `///` immediately above: use awk/python to detect 2-line window
- Compute coverage % per crate
- Target ≥80%; flag below as P1

**DOCS-02 Module-level docs:**
- For each .rs file: check first 5 lines for `//!` doc comment
- `find crates -name "*.rs" | while read f; do head -5 "$f" | grep -q "^//!" || echo "$f"; done`
- List files missing module-level docs

**DOCS-03 Stale comments:**
- `grep -rnE "// TODO|// FIXME|// XXX|// HACK" crates/ | head -50`
- `grep -rnE "// NOTE: legacy|// old|deprecated" crates/ | head -30`
- Look for references to old modules/APIs that no longer exist
- Cross-reference with git log for files that haven't been touched in 6+ months

**DOCS-04 External doc accuracy:**
- README.md: list claims, check each against current code (e.g., "Supports X models" → check arch registry)
- AGENTS.md: check naming/structure claims against current codebase
- .planning/PROJECT.md, REQUIREMENTS.md: cross-check against current state
- Flag outdated claims

**DOCS-05 ADRs:**
- `find . -path "*/adr/*" -o -path "*/ADR*" -o -path "*/decisions/*"` 
- If exists, list ADRs and their topics
- Identify architectural decisions made but NOT documented (e.g., "Why vllm-dist is unused?", "Why self-spec uses 1/8 layers?")
- Check commit messages for rationale not in ADRs
</action>
