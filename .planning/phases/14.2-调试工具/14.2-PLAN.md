## Implementation Details

### Debug Endpoints
- `GET /debug/metrics` - Returns current metrics as JSON
- `GET /debug/kv-cache` - Returns KV cache state (blocks, usage, prefix cache)
- `GET /debug/trace` - Returns tracing status

### Request Tracing
Added hierarchical tracing spans:
- `scheduler.add_request` - Request arrival with request_id, prompt_len
- `scheduler.build_batch` - Batch building with waiting/running counts
- `scheduler.update` - Update processing with seq/token counts
- `scheduler.decode_token` - Per-token decoding with seq_id, token
- `scheduler.prefill` - Prefill phase transitions
