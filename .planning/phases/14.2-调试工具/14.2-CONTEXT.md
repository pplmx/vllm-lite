# Phase 14.2: 调试工具 - Context

**Gathered:** 2026-04-27
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement developer debugging utilities: request tracing via tracing spans, KV cache dump endpoint, and metrics snapshot endpoint.

</domain>

<decisions>
## Implementation Decisions

### Tracing Architecture
- Use existing tracing infrastructure (tracing crate)
- Add hierarchical spans for: request → prefill → decode → token generation
- Spans include request_id, seq_id, phase, timing

### KV Cache Dump
- Add `/debug/kv-cache` endpoint to server
- Returns block contents, prefix match status, hit rate
- JSON format for easy parsing

### Metrics Snapshot
- Add `/debug/metrics` endpoint to server
- Returns current metrics as JSON
- Includes queue depth, active sequences, cache stats

</decisions>
