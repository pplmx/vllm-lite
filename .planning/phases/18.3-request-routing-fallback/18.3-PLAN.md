# Phase 18.3 Plan: Request Routing + Fallback

## Requirements

- RTE-01: Request can specify `draft_model_id` via `SamplingParams` or `Request` struct
- RTE-02: Scheduler routes request to correct draft model instance during batch composition
- RTE-03: Multiple drafts can coexist in the same batch (mixed draft routing across requests)
- FALL-01: External draft load failure → automatic fallback to self-spec path (v17 baseline)
- FALL-02: Runtime draft inference error → graceful degradation to non-speculative decode for that request

### 2. Create draft resolver

**`crates/core/src/speculative/draft_resolver.rs`:**
```rust
use std::sync::{Arc, Mutex};
use vllm_traits::ModelBackend;

#[derive(Clone)]
pub(crate) enum ResolvedDraft {
    External(Arc<Mutex<Box<dyn ModelBackend>>>),
    SelfSpec(Arc<Mutex<Box<dyn ModelBackend>>>),
    None,
}

impl ResolvedDraft {
    pub fn is_some(&self) -> bool {
        !matches!(self, ResolvedDraft::None)
    }
    pub fn kind(&self) -> &'static str {
        match self {
            ResolvedDraft::External(_) => "external",
            ResolvedDraft::SelfSpec(_) => "self_spec",
            ResolvedDraft::None => "none",
        }
    }
}

pub trait DraftLoader: Send + Sync {
    fn load(&self, id: &DraftId) -> std::result::Result<Box<dyn ModelBackend>, String>;
}

pub(crate) struct DraftResolver {
    registry: Arc<DraftModelRegistry>,
    self_spec: Option<Arc<Mutex<Box<dyn ModelBackend>>>>,
    loader: Arc<dyn DraftLoader>,
    metrics: Arc<EnhancedMetricsCollector>,
}

impl DraftResolver {
    pub fn new(
        registry: Arc<DraftModelRegistry>,
        self_spec: Option<Arc<Mutex<Box<dyn ModelBackend>>>>,
        loader: Arc<dyn DraftLoader>,
        metrics: Arc<EnhancedMetricsCollector>,
    ) -> Self { ... }

    /// Resolve a draft for a request. Returns SelfSpec on any failure (FALL-01).
    pub fn resolve(&self, request_draft_id: Option<&DraftId>) -> ResolvedDraft {
        let id = match request_draft_id {
            None => {
                // No external draft requested
                self.metrics.inc_draft_resolution("none");
                return match &self.self_spec {
                    Some(s) => ResolvedDraft::SelfSpec(s.clone()),
                    None => ResolvedDraft::None,
                };
            }
            Some(id) => id,
        };

        // Try already-loaded
        if self.registry.is_loaded(id) {
            // Caller still needs an Arc<Mutex<Box<dyn ModelBackend>>>;
            // for v18.3 we re-resolve via a registry helper that yields a clone.
            if let Some(backend) = self.registry.get_loaded_backend(id) {
                self.metrics.inc_draft_resolution("external");
                return ResolvedDraft::External(backend);
            }
        }

        // Try to load it (lazy) — FALL-01 path on failure
        match self.loader.load(id) {
            Ok(backend) => {
                let arc_backend: Arc<Mutex<Box<dyn ModelBackend>>> =
                    Arc::new(Mutex::new(backend));
                if self.registry.attach_loaded_budgeted(id, ...).is_err() {
                    // attach failed (budget/etc.) — fall back
                    self.metrics.inc_draft_load_failure();
                    return self.fallback_to_self_spec();
                }
                // Register the backend so future lookups hit the cache
                // (We need a separate "attach with pre-built Arc" method or we
                // re-extract from the registry — see step 4.)
                self.metrics.inc_draft_resolution("external");
                ResolvedDraft::External(/* get from registry */)
            }
            Err(e) => {
                tracing::warn!(draft_id = %id, error = %e, "draft load failed; falling back to self-spec");
                self.metrics.inc_draft_load_failure();
                self.fallback_to_self_spec()
            }
        }
    }

    fn fallback_to_self_spec(&self) -> ResolvedDraft {
        match &self.self_spec {
            Some(s) => ResolvedDraft::SelfSpec(s.clone()),
            None => ResolvedDraft::None,
        }
    }
}
```

### 5. Runtime error handling (FALL-02)

**Wrap per-request draft forward in error-catching helper:**
```rust
fn run_draft_with_fallback(
    backend: &Arc<Mutex<Box<dyn ModelBackend>>>,
    input: &ModelInput,
) -> Result<ModelOutput, ModelError> {
    let mut guard = backend.lock().expect("backend mutex poisoned");
    guard.forward(input)  // returns Result; we propagate
}
```

In the step loop, if `run_draft_with_fallback` returns `Err`, we log,
increment `draft_runtime_errors_total`, and fall back to non-spec decode
for that request (this step and all subsequent steps until completion).
Other requests in the batch are unaffected.

To avoid constant re-attempts that will keep failing, mark the request as
`degraded_draft = true` in its `Sequence` and short-circuit future calls.

## Success Criteria

1. ✓ `Request::new(...).draft_model_id == None`
2. ✓ `Request::with_draft_model(id).draft_model_id == Some(id)`
3. ✓ When draft is loaded and request names it: spec decode runs with that draft
4. ✓ When loader fails (unknown id, budget exceeded, model error): silently falls back to self-spec, no panic
5. ✓ When draft forward errors at runtime: that specific request degrades to non-spec decode, others unaffected
6. ✓ Two requests in one batch with different drafts each get their own draft (mixed routing)
7. ✓ Metrics counters increment correctly
8. ✓ All 254+ tests pass; clippy/fmt clean
