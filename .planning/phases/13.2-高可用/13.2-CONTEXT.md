# Phase 13.2: 高可用 - Context

**Gathered:** 2026-04-27
**Status:** Ready for planning
**Mode:** Autonomous

## Phase Boundary

High availability with leader election, automatic failover, Prometheus metrics, HPA support, and consistent hash routing.

## Requirements

- HA-01: Leader election using K8s Lease API
- HA-02: Automatic failover when leader node becomes unavailable
- HA-03: PodDisruptionBudget for safe evictions during cluster upgrades
- HA-04: In-flight request preservation during failover
- OPS-03: Prometheus metrics endpoint (/metrics)
- OPS-04: HPA (Horizontal Pod Autoscaler) support based on queue depth
- CLUSTER-04: Consistent hash routing for request distribution

## Success Criteria

1. Kill leader pod, follower takes over within 30s
2. In-flight requests complete or are retried
3. HPA scales replicas based on queue depth
4. `/metrics` exposes request_count, queue_depth, gpu_memory_used
5. Consistent hash routes same prompt to same node
6. PodDisruptionBudget allows max 1 unavailable during upgrade

## Codebase Context

Existing infrastructure:
- `crates/server/src/main.rs` — Already has /metrics endpoint
- `crates/core/src/engine.rs` — Scheduler engine
- `crates/server/src/health.rs` — Health checker

New components needed:
- Leader election via K8s Lease API
- Request tracking for failover
- Consistent hash routing in router
