# Phase 13.2 Plan: 高可用

**Phase:** 13.2
**Created:** 2026-04-27
**Status:** Planned

## Task Breakdown

### Task 1: Prometheus Metrics Enhancement (OPS-03)
- Add `request_count` counter
- Add `queue_depth` gauge
- Add `gpu_memory_used` gauge
- Add `active_requests` gauge
- Expose via existing /metrics endpoint

### Task 2: HPA Support (OPS-04)
- Add custom metrics for HPA
- Configure HPA in Helm chart to scale on queue_depth
- Add VPA configuration option

### Task 3: Leader Election (HA-01)
- Create K8s Lease-based leader election
- Use `coordination.k8s.io/v1` Lease API
- Implement leader election in scheduler
- Only leader processes scheduling decisions

### Task 4: Automatic Failover (HA-02)
- Detect leader failure via Lease expiry
- Promote follower to leader
- Update cluster state
- Log failover events

### Task 5: In-flight Request Preservation (HA-04)
- Track active requests in flight
- On failover, complete or route requests
- Implement request timeout handling
- Add request deduplication

### Task 6: Consistent Hash Routing (CLUSTER-04)
- Implement consistent hash for request routing
- Use prompt hash as routing key
- Route to node with cached KV
- Fallback to least-loaded node on cache miss

### Task 7: PodDisruptionBudget (HA-03)
- Already included in Phase 13.1 Helm chart
- Verify configuration allows max 1 unavailable

## Dependencies
