Spawn API audit subagent:

**API-01 Public API consistency:**
- For each crate, list `pub fn` signatures: `grep -rhE "^pub fn|^pub async fn" crates/$CRATE/src | sort`
- Check builder pattern consistency: `grep -rh "Builder::new\|::default()" crates/ | sort | uniq -c | sort -rn | head -20`
- Check parameter ordering conventions (e.g., `&self` first, then args; or args first?)
- Check return type consistency (Result vs panic)
- Flag inconsistent patterns

**API-02 Error types:**
- For each crate, find error enums/structs: `grep -rhE "^pub enum.*Error|^pub struct.*Error" crates/$CRATE/src`
- Check thiserror usage: `grep -rhE "use thiserror" crates/$CRATE/src | sort | uniq -c`
- For each error enum, check:
  - Variants cover common cases (NotFound, Invalid, IO, Internal)
  - Error messages include relevant context (file path, line, ID)
  - Use of structured fields (`#[source]`, `#[from]`)
- Flag errors without thiserror, generic String errors, missing variants

**API-03 Error ergonomics:**
- Count `.unwrap()` in non-test code: `grep -rE "\.unwrap\(\)" crates/ --include="*.rs" | grep -v "test\|tests/" | wc -l`
- Count `.expect()` similarly
- Find missing `From` impls: errors that should auto-convert via `?` but don't
- Find missing `.context()` calls: errors that lose context when propagated

**API-04 Trait design:**
- List all public traits: `grep -rhE "^pub trait" crates/`
- For each, check:
  - Object safety (no generic methods, no `Self` in return)
  - Async/sync consistency (does the trait mix async and sync methods?)
  - Default methods provided
  - `dyn Trait` usage: `grep -rE "dyn [A-Z][a-zA-Z]+" crates/`
- Flag non-object-safe public traits
- Flag mixed sync/async traits

**API-05 Deprecation hygiene:**
- `grep -rnE "#\[deprecated" crates/`
- List deprecated items without migration path (no comment explaining replacement)
- Check for items removed but still mentioned in docs (cross-ref Phase 22 docs audit)
