# Phase 15.1: FlashAttention V3 - Context

**Phase:** 15.1
**Gathered:** 2026-04-27
**Status:** Ready for planning

---

## Phase Boundary

Implement FlashAttention V3 kernel with improved memory efficiency and MQA/GQA support.

---

## Success Criteria

1. FA-V3 kernel implemented with causal masking and softmax scaling
2. MQA (Multi-Query Attention) support for models like Llama 3
3. GQA (Grouped-Query Attention) support for Gemma2, Mistral, Llama 4
4. Benchmark shows >30% speedup vs current FlashAttention V2

---

## Implementation Decisions

### Kernel Architecture
- FA-V3 uses online softmax with reduced VRAM usage
- Support for sliding window attention (SWA) in Gemma3
- Backward pass for training scenarios

### MQA/GQA Support
- Models: Llama 3 (MQA), Gemma3/Mistral/Llama 4 (GQA)
- Unified attention interface supporting 1, 4, or 8 KV heads

### Integration with vllm-lite
- Replace existing FlashAttention V2 kernels
- Maintain CUDA Graph compatibility
- Benchmark comparison harness

---

## Codebase Context

Existing FlashAttention V2 in `crates/model/src/components/attention/gqa.rs` with `flash_attn_v2` feature flag. New FA-V3 should be additive with a `flash_attn_v3` feature flag for gradual rollout.
