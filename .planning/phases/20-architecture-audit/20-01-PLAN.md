<action>
Spawn a single architecture audit subagent. Pass it:
- Phase goal and CONTEXT.md
- The 5 ARCH-* requirement IDs
- Concrete methodology:
  1. `cargo metadata --format-version 1` → JSON dependency graph → render directional flow table (rows=source crate, cols=target crate)
  2. For each crate, list all modules with `find crates/$CRATE/src -name "*.rs"` and a one-line single-responsibility statement per module
  3. Detect God modules: `find crates -name "*.rs" | xargs wc -l | awk '$1 >= 1000'` and `grep -c "^pub " file.rs` for pub-item count
  4. Circular dependency scan: build graph from cargo metadata, detect cycles with DFS
  5. Layering matrix: rows=crate, cols={traits, core, model, server, dist, testing, benches}; mark allowed inbound per `traits ← core ← {model, server, dist}` rule
  6. Test architecture: `find . -name "tests" -type d`, list unit (in `src/`) vs integration (in `tests/`) vs bench (in `benches/`) boundaries; check `vllm-testing` reuse

Output two files:
- REPORT.md: detailed sections per topic
- SUMMARY.md: P0/P1/P2 prioritized table with columns | ID | Description | Severity | Source | Estimated effort |

The subagent must NOT modify any source files. Verify with `git diff --stat` at end.
</action>
<verify>

</verify>
