# Phase 17.4: Speculative Warmup & Metrics - Context

**Gathered:** 2026-05-13
**Status:** Ready for planning

<domain>

## Phase Boundary

Draft KV cache is populated during/after target prefill so the first speculative step has valid draft state. Comprehensive spec decode metrics track acceptance rate, efficiency, and speedup ratio vs non-speculative baseline.

Key deliverables:

- Draft model's KV cache is populated during/after target prefill
- Speculative efficiency (draft tokens / total tokens) reported
- Throughput speedup ratio vs non-speculative baseline reported
- Acceptance rate tracked per-request and aggregated via Prometheus
</domain>

<decisions>

## Implementation Decisions

### Warmup (WARM-01, WARM-02)

- Populate draft KV cache after each prefill step — engine's process_output runs after forward, at that point target KV state is available; run a draft forward to fill draft KV
- Only for sequences in speculative mode — non-speculative sequences skip warmup
- Wire warmup in Engine::step() after prefill detection (check batch.phase == Prefill), call draft_model.lock().unwrap().forward_to_layer(...) with draft KV block IDs

### Metrics (MTRC-01, MTRC-02, MTRC-03)

- Add speculative_efficiency (draft_tokens / total_tokens) to EnhancedMetrics as an atomic counter
- Add throughput_speedup_ratio to EnhancedMetrics (requires comparing non-spec baseline, computed offline in benchmark)
- Track acceptance rate per-request via existing record_speculative_draft() and aggregate via Prometheus

### the agent's Discretion

Exact integration point in Engine::step() for warmup, metric naming conventions, and Prometheus label design are left to the agent's discretion.

</decisions>

<specifics>

## Specific Ideas

- Warmup after prefill: `if batch.phase == Prefill { draft_model.forward_to_layer(...) }` — this runs the draft model on the same input, populating its KV cache (with upto_layer = draft_layer_count)
- Draft model `forward_to_layer()` with the draft KV block IDs ensures the draft's PagedKvCache gets populated without affecting the target's cache
- Efficiency = draft_tokens_generated / (draft_tokens_generated + target_tokens_generated) — higher is better
- Speedup ratio needs to be computed offline by comparing benchmark results: spec_tokens_per_sec / non_spec_tokens_per_sec

</specifics>
