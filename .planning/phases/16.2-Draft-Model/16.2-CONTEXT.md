# Phase 16.2: Draft Model - Context

**Gathered:** 2026-04-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement self-speculation using the same model with reduced layer count. The draft model shares weights with the target model (zero copy), uses configurable layer count per architecture, and supports adaptive KV cache reuse.

</domain>

<decisions>
## Implementation Decisions

### Self-Speculation Approach
- Use first N layers of target model as draft (not a separate model)
- Share weight tensors with target (zero copy memory)
- Draft layer count configurable per architecture in config.json

### KV Cache Reuse
- Accepted draft KV blocks appended to target KV cache
- No recomputation of accepted prefixes

### Layer Configuration
Default draft layer counts per architecture:
- Llama: 4 layers (for 32-layer model) = 1/8 of layers
- Qwen: 4 layers
- Mistral: 4 layers
- Gemma: 4 layers

Configurable via `SpeculationConfig.draft_layers` or architecture config.

### the agent's Discretion
Implementation details at agent's discretion. Follow existing patterns.

</decisions>
