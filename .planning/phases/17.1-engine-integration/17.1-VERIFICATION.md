# Phase 17.1 Verification: Engine Integration

## Success Criteria Verification

| # | Criterion | Status | Evidence |
|---|-----------|--------|----------|
| 1 | Engine dispatches through single unified `step(max_draft)` method | ✓ | `step_with_draft(Option<usize>)` dispatches to `step_internal()` or `step_speculative_inner(n)` (speculative.rs:13-18) |
| 2 | Draft tokens generated in batched per-position forward pass | ✓ | `generate_batched_drafts()` iterates positions 0..max_draft, builds per-position batch, single forward per position (speculative.rs:167-241) |
| 3 | Token verification uses logit-based probability comparison | ✓ | `verify_draft_tokens_logits()` uses `forward_logits()` + argmax comparison (speculative.rs:246-330) |

| 6 | Speculative path falls back gracefully on draft model error | ✓ | Draft generation errors caught → warn → fallback to `step_internal()` (speculative.rs:78-83); no draft model → warn → empty drafts (speculative.rs:177-180) |
