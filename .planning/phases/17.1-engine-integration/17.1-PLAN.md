# Phase 17.1 Plan: Engine Integration

**Created:** 2026-05-13
**Status:** Planned
**Requirements:** ENG-01, ENG-02, ENG-03, ENG-04, ENG-05, ENG-06
**Success Criteria:** 6/6

## Summary

Wire the v16.0 speculative decode architecture (DraftVerifier, SpeculativeModel, SpeculationConfig) into the main Engine inference loop. Replace per-sequence draft generation with batched per-position forward passes, implement logit-based verification, add KV cache rollback for rejected drafts, and unify all decode paths through a single `step(max_draft)` entry point.

## Plans

### Plan 17.1-A: Unified `step()` Entry Point

**Files:** `crates/core/src/engine.rs`, `crates/core/src/engine/speculative.rs`
**Requires:** ENG-01, ENG-05
**Depends on:** None

Refactor `Engine::run()` to dispatch through a single `step(max_draft: Option<usize>)` method. The inner `step_speculative()` becomes `step_speculative_inner()`. The `step_speculative()` and `step_adaptive_speculative()` public methods remain for backward compatibility but delegate to the unified path.

- Replace the if-else chain in `Engine::run()` with `self.step(None)` (non-speculative by default)
- Add `step(max_draft: Option<usize>)` that checks `speculative_mode` and dispatches:
  - `None` or `Some(0)` → non-speculative step (existing `step()`)
  - `Some(n)` → `step_speculative_inner(n)`
- Rename current `step_speculative()` body to `step_speculative_inner(max_draft: usize)`
- On draft model error in `step_speculative_inner`: catch, log warning, fall through to non-speculative `step()`
- Wire `step_adaptive_speculative()` through the same `step()` entry point using `adaptive_decoder.current_max_draft_tokens()`

### Plan 17.1-B: Batched Draft Generation

**Files:** `crates/core/src/engine/speculative.rs`
**Requires:** ENG-02
**Depends on:** Plan 17.1-A

Refactor `generate_draft_tokens()` from per-sequence loop to batched per-position forward pass. All sequences generate draft position k before advancing to k+1.

- Replace `generate_draft_tokens_with_limit()` body:
  - Build per-position batches: for `pos` in `0..max_draft`, collect all sequences' current input tokens
  - Call `draft_model.forward()` once per position across all sequences
  - Collect output tokens per sequence per position
- Handle padding: short sequences produce fewer drafts but participate in every position batch
- Pre-allocate `Vec<Vec<TokenId>>` output with capacity for all sequences
- Track position indices per-sequence for the next iteration's input tokens
- Preserve existing error handling: catch draft model error → log warning → return empty drafts

### Plan 17.1-C: Logit-Based Verification

**Files:** `crates/core/src/engine/speculative.rs`, `crates/core/src/speculative/verifier.rs`
**Requires:** ENG-03
**Depends on:** Plan 17.1-B

Replace exact token match verification with logit-based probability comparison using `forward_logits()`. Wire the existing `DraftVerifier` trait's `verify()` method with probability-based rejection sampling.

- Add `ModelBackend::forward_logits()` method that returns logits for each position
- In `verify_draft_tokens()`: concatenate `[input_tokens | draft_tokens]` per sequence, call `forward_logits()`, compare draft_token vs target_token probabilities using rejection sampling: `r ~ Uniform(0,1)`, accept if `r < min(1, target_prob / draft_prob)`
- For greedy decoding (temperature=0): use argmax comparison (accept if both models predict the same token)
- Fall back to exact match if probabilities are unavailable
- Wire `DraftVerifier::verify()` from `verifier.rs` into the verification loop

### Plan 17.1-D: KV Cache Rollback for Rejected Drafts

**Files:** `crates/core/src/scheduler/memory/allocator.rs` (or relevant MemoryManager), `crates/core/src/engine/speculative.rs`
**Requires:** ENG-04
**Depends on:** Plan 17.1-C

When draft tokens are rejected, roll back the KV cache blocks allocated for those positions so they don't leak into subsequent steps.

- Add `rollback(seq_id: SeqId, num_tokens: usize)` to `MemoryManager`:
  - Compute how many KV blocks to free based on `num_tokens` and block size
  - Decrement computed token count for the sequence
  - Return freed blocks to the free pool
- Call `memory_manager.rollback(seq_id, rejected_count)` after verification for each sequence that had rejected drafts
- Update `batch.num_computed_tokens[i]` to reflect the rolled-back state
- Only rollback the KV blocks for the draft model's forward — target model's KV cache is authoritative

### Plan 17.1-E: Scheduler Multi-Token Input Tracking

**Files:** `crates/core/src/engine/speculative.rs`, `crates/core/src/scheduler/engine.rs`
**Requires:** ENG-06
**Depends on:** Plan 17.1-C

The scheduler's `update()` method already accepts `input_counts: &[usize]` (supports values > 1). Wire the actual accepted draft count per sequence into this parameter instead of hardcoding `vec![1; tokens.len()]`.

- Count accepted tokens per sequence after verification (includes bonus token)
- Pass the actual counts as `input_counts` to `scheduler.update()`
- Verify that scheduler position tracking, block allocation, and prefix caching work correctly with multi-token inputs
- Update `num_computed_tokens` in the batch accordingly

### Plan 17.1-F: Tests for Speculative Engine Integration

**Files:** `crates/core/src/engine/speculative.rs` (add `#[cfg(test)]` module), `tests/` (integration test)
**Requires:** ENG-01 through ENG-06
**Depends on:** Plans 17.1-A through 17.1-E

Add unit tests and integration tests covering the speculative engine integration.

- **Unit tests:**
  - `test_step_unified_dispatch` — verify `step(None)` vs `step(Some(5))` dispatches correctly
  - `test_batched_draft_generation` — verify batched per-position output matches expected shape
  - `test_logit_verification_exact_match` — verify greedy-mode exact match verification
  - `test_logit_verification_rejection_sampling` — verify probability-based acceptance
  - `test_kv_rollback_rejected_drafts` — verify MemoryManager rollback counts
  - `test_scheduler_multi_token_update` — verify `input_counts > 1` is accepted
  - `test_draft_model_error_fallback` — verify graceful fallback on draft error
- **Integration tests:**
  - `test_speculative_step_produces_output` — end-to-end speculative step with `FakeModel`
  - `test_speculative_vs_non_speculative_equivalence` — same input yields same output (modulo accepted bonus tokens)

## Risk Assessment

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| KV cache rollback frees wrong blocks | Silent corruption | Medium | Unit test with known state |
| Batched draft generation with different sequence lengths | Off-by-one errors | Medium | Handle as padded + mask |
| `forward_logits()` trait change breaks existing models | Compile errors | Low | Default impl returning `forward()` tokens |
| Draft model error during batched forward | Partial batch failure | Low | Catch per-position, log warning, continue with empty drafts |
