## Requirements Coverage

**Total:** 21 of 21 requirements addressed (100%).

### Phase 36 — Critical Bug Fixes (3 reqs)

- OPS-02: Engine::step() speculative-mode hang — ✅ **Root cause**:
  DashMap shard re-entry in `EnhancedMetricsCollector::record_per_request_acceptance`
  (entry guard held across `len()` call). **Fix**: scope the entry to
  release the shard lock before calling `len()`. The 2 `#[ignore]`'d
  Phase 19 e2e tests in `engine_wiring.rs` now pass; 7 `#[ignore]`'d
  unit tests in `engine/spec_dispatch/tests.rs` also unblocked.
- OPS-03: 10 cargo doc broken-link warnings — ✅ All closed
  (`crates/testing/src/lib.rs`, `crates/core/src/engine.rs`,
  `crates/core/src/speculative/registry/{mod,lifecycle}.rs`,
  `crates/model/src/components/{attention/mod,block,decoder_block/mod}.rs`).

## Tech Debt Acknowledged

The following items remain open and are tracked as v23.0 candidates:

1. **ARF-06 partial**: `engine.rs` (1057 LOC) was partially split in
   v20.0 (`engine/spec_dispatch/` sub-tree). The remaining file
   consolidates Engine struct + impl block + CUDA Graph wrappers +
   beam search + request/response handling. A full
   single-responsibility split is feasible but requires careful test
   coverage to avoid regression; deferred to v23.0+.

2. **Trait-object lock types**: `Arc<Mutex<Box<dyn ModelBackend>>>`
   in `engine.rs` and `engine/spec_dispatch/warmup.rs` are
   `std::sync::Mutex`, not `parking_lot::Mutex`. Migrating would
   change the public trait-object signature and require a coordinated
   `vllm-traits` API change. Out of scope for v22.0 polish.

3. **Other `.to_lowercase()` sites**: `crates/model/src/phi4/arch.rs`
   and `crates/model/src/mistral_small/arch.rs` still use
   `.to_lowercase().contains(...)` / `.starts_with(...)` patterns.
   These are not in the per-load hot path and require a small helper
   to migrate cleanly; deferred to v23.0 maintenance cycle.

4. **Other v22.0 deferred items** (carried forward from v21.0 close):
   - Long context (>32K) — NMC-01
   - Multimodal/Vision — NMC-02
   - Tool calling — NMC-03
   - Doc coverage 97.8% → 99%+ — RFU-06
   - Multi-node / vllm-dist resurrection — OPS-05
   - Real-model benchmark — OPS-04 (requires GPU env)
