# Phase 25-04: Convert `CudaGraphError` to thiserror + From shim — API-F-03

### Task 3: Add `From<&str>` and `From<String>` convenience impls (semver compat shim)

The original enum construction sites used `CudaGraphError::CaptureFailed("…".to_string())` and similar — these still work because each variant takes `String`. To allow ergonomic construction from `&str` literals (matching the spirit of the original API), add explicit `From` impls:

```rust
// In crates/model/src/kernels/cuda_graph.rs after the enum
impl CudaGraphError {
    /// Convenience constructor from a string literal.
    pub fn capture_failed(msg: impl Into<String>) -> Self {
        Self::CaptureFailed(msg.into())
    }

    pub fn launch_failed(msg: impl Into<String>) -> Self {
        Self::LaunchFailed(msg.into())
    }

    pub fn invalid_node(msg: impl Into<String>) -> Self {
        Self::InvalidNode(msg.into())
    }

    pub fn unsupported(msg: impl Into<String>) -> Self {
        Self::Unsupported(msg.into())
    }
}
```

**Note:** The constructors above are *not* required for correctness — the enum works without them. They preserve ergonomics for any code that did `CudaGraphError::CaptureFailed("msg".to_string())` by allowing `CudaGraphError::capture_failed("msg")`.

If call sites already use the variant constructor directly (`CudaGraphError::CaptureFailed(format!(…))`), no shim is needed.
