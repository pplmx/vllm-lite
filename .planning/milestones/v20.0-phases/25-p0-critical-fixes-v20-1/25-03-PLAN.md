# Phase 25-03: Convert `ModelError` struct → enum + migrate match sites — API-F-01

## Objective

Convert `ModelError` from a wrapper struct (single `String` field) into a structured `#[non_exhaustive]` enum with variants (`Message`, `Backend`, `Config`, `Candle`, `Io`) so callers can pattern-match on failure modes, and migrate any existing `match ModelError { … }` sites to enum variants.

## Files to Modify

| File | Lines | Change |
|------|------:|--------|
| `crates/traits/src/model.rs` | 1-25 | Replace wrapper struct with `#[non_exhaustive] pub enum ModelError` (already done in commit `2ffd4af`); verify `From<candle_core::Error>` and `From<std::io::Error>` impls use new variants |

**Audit references:** `.planning/audit/api/REPORT.md` §2.2 `ModelError`; API-02-F01 (P0); `.planning/audit/architecture/REPORT.md` ARCH-F-13 cross-reference for `TensorParallelError`.

## Files to Create

None — `ModelError` is `#[non_exhaustive]` so adding new variants in the future is non-breaking.

## Tasks

### Task 1: Confirm `ModelError` enum shape

Read `crates/traits/src/model.rs` and verify:

```bash
grep -A20 "pub enum ModelError" crates/traits/src/model.rs
```

Expected variants (all `#[error("...")]`-formatted):
- `Message(String)` — generic free-form message
- `Backend(String)` — backend-specific error
- `Config(String)` — configuration error
- `Candle(#[from] candle_core::Error)` — candle-core error chain (only when `feature = "candle"`)
- `Io(#[from] std::io::Error)` — I/O error chain

Plus:
- `#[non_exhaustive]` attribute (so adding variants in v20.x is non-breaking for downstream consumers)
- `From<candle_core::Error>` impl preserved via `#[from]` attribute on the `Candle` variant
- `From<std::io::Error>` impl preserved via `#[from]` attribute on the `Io` variant
- `ModelError::new(msg)` constructor still returns `Self::Message(msg.into())` for backward compat

If any of the above is missing, add it before proceeding.

### Task 2: Audit existing `match ModelError` sites

```bash
# Find every site that pattern-matches ModelError (should be 0 for exhaustive matching,
# because #[non_exhaustive] forbids exhaustive matches from outside this crate)
grep -rn "match.*ModelError\|ModelError::" crates/ --include="*.rs" | grep -v "/lib.rs\|/model.rs" | head -30
```

For each external site that uses `match` on `ModelError`, decide:
1. **No match needed:** site only constructs (`ModelError::new(...)`) — no change.
2. **Structural conversion:** site constructs via `ModelError::new("…")` — consider migrating to `ModelError::Message(…)` or a more specific variant (`Backend`, `Config`).
3. **Exhaustive match:** site does `match model_err { … }` — add a `_ =>` wildcard arm to be future-proof against new variants.

If exhaustive matches are found in `crates/core/`, `crates/model/`, or `crates/server/`, migrate them per §2 below.

### Task 3: Migrate `match` sites to enum variants (if any)

For each `match` site identified in Task 2, refactor:

```rust
// Before (with struct ModelError { message: String }):
match err {
    ModelError { message } => handle_message(message),
}

// After (with enum ModelError):
match err {
    ModelError::Message(msg) => handle_message(msg),
    ModelError::Backend(msg) => handle_backend(msg),
    ModelError::Config(msg) => handle_config(msg),
    ModelError::Candle(e) => handle_candle(e),
    ModelError::Io(e) => handle_io(e),
    #[allow(unreachable_patterns)] // for #[non_exhaustive]
    other => handle_default(other),
}
```

## Rollback

1. `git revert` this plan's commit.
2. Restore `crates/traits/src/model.rs` to the wrapper struct (the v19.0 state).
3. Add a type alias shim if downstream code relied on the new variants: `pub type ModelError = OldModelError;`
4. Re-run baseline test suite.

**Backward-compat note:** The conversion from struct to `#[non_exhaustive]` enum is **breaking** for downstream code that does exhaustive matching. The `#[non_exhaustive]` attribute forces a wildcard arm at all match sites, providing the migration path. The `ModelError::new` constructor is preserved (returns `Self::Message`), so callers that constructed the old struct still compile.

If rollback is needed, `pub use` re-exports in any downstream consumer may need to be re-pointed at the old struct via a type alias.
