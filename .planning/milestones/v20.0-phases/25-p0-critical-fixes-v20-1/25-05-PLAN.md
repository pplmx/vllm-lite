### Task 3: Strengthen the `dyn_safety.rs` pattern (optional, hardening)

The current pattern (`let _ = std::any::type_name::<dyn Trait>()`) only proves the trait name can be mentioned; it does not verify that all required methods can be called on the trait object. To make the test more robust, augment with method existence checks:

```rust
// Example for Architecture:
fn assert_architecture_obj_safe(t: &dyn Architecture) {
    let _ = t.name();
    let _ = t.capabilities();
}
```

However, **do not** add this if it requires trait method signatures that may change. The current pattern is the minimum-viable compile-only check.
