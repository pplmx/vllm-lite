# Phase 25-01: Eliminate `vllm-model → vllm-dist` edge (feature-gate) — ARCH-F-11

## Objective

Eliminate the `vllm-model → vllm-dist` sibling-tier dependency edge (ARCH-F-11) by feature-gating every `vllm-dist` import site behind `--features multi-node`, so `cargo build -p vllm-model --no-default-features` no longer pulls `vllm-dist` into its dependency tree.

| `crates/model/src/qwen3/mod.rs` | 7-8 | Verify `#[cfg(feature = "multi-node")] pub mod tp;` (already done) |

### Task 2: Close the `tp.rs` feature-gate gap

The `qwen3/tp.rs` module is gated by `#[cfg(feature = "multi-node")] pub mod tp;` in `qwen3/mod.rs:7-8`, but its `use vllm_dist::TensorParallelConfig;` at line 8 is not feature-gated. While the module-level gate prevents compilation under default features, hardening the `use` line itself makes the gating intent explicit and matches the pattern used in `block.rs` and `model.rs`.

Edit `crates/model/src/qwen3/tp.rs` to add the cfg attribute:

```rust
// Before (line 7-8):
use candle_core::{Device, Result as CandleResult};
use vllm_dist::TensorParallelConfig;

// After:
use candle_core::{Device, Result as CandleResult};
#[cfg(feature = "multi-node")]
use vllm_dist::TensorParallelConfig;
```

**Why:** Defensive gating — guarantees that even if a future refactor un-gates `pub mod tp;`, the `use` won't silently re-introduce the dep edge. Matches the pattern in `block.rs:11` and `model.rs:8`.
