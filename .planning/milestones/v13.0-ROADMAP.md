## Milestone Summary

### Key Decisions

- Decision: Use K8s native APIs (Lease API for leader election) — Rationale: Simpler than external consensus
- Decision: Helm chart + Operator hybrid — Rationale: Helm for basic deploy, Operator for complex lifecycle
- Decision: gRPC for inter-node communication — Rationale: Efficient binary protocol, good Rust support

### Issues Resolved

- Resolved: Multi-node discovery without service mesh
- Resolved: Leader election for single-node and multi-node scenarios
- Resolved: Request correlation across HA failover

### Issues Deferred

- K8S-02: Full Go Kubernetes Operator (scaffolded only)
- gRPC multi-node testing (requires actual K8s cluster)
- Production Lease API integration (in-memory state for dev)
- Multi-node failover coordination
- TLS integration with axum server
- JWT library integration

### Technical Debt Incurred

- K8S-02: Full Go Kubernetes Operator not implemented
- gRPC multi-node testing deferred until cluster available
- Leader election uses in-memory state for dev
- TLS termination scaffolded, not fully integrated
- JWT validation stubbed
