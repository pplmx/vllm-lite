# Phase 40 Plan 01: Convert TensorParallelError to thiserror

**Goal:** Replace manual `Display`/`std::error::Error` impls on `TensorParallelError`
(traits/src/types.rs:86-112) with `#[derive(thiserror::Error)]` matching the project's
19 other error enums. All call sites compile unchanged; 6 variants preserved.

<acceptance_criteria>

   `InputSizeMismatch`, `AllReduceFailed(String)`, `CudaError(String)`) has an
   `#[error("...")]` attribute matching the existing `Display` strings
3. Manual `impl Display for TensorParallelError` and
   `impl std::error::Error for TensorParallelError` are removed

</acceptance_criteria>
<action>
1. Open `crates/traits/src/types.rs` and locate the `TensorParallelError` enum (lines 86-112)
2. Replace the `#[derive(Debug, Clone)]` attribute with `#[derive(Debug, Clone, thiserror::Error)]`
3. Add `#[error("...")]` to each of the 6 variants, copying the strings verbatim from the existing `impl Display`:
   - `InvalidWorldSize` → `#[error("World size must be > 0")]`
   - `InvalidRank` → `#[error("Rank must be < world size")]`
   - `DeviceMismatch` → `#[error("Number of device IDs must match world size")]`
   - `InputSizeMismatch` → `#[error("Input size does not match expected size per rank")]`
   - `AllReduceFailed(String)` → `#[error("All-reduce failed: {0}")]`
   - `CudaError(String)` → `#[error("CUDA error: {0}")]`
4. Delete the manual `impl std::fmt::Display for TensorParallelError` block (lines 97-110)
5. Delete the `impl std::error::Error for TensorParallelError {}` line (line 112)
6. Verify `thiserror` is in `crates/traits/Cargo.toml` `[dependencies]` (it should already be,
   since other project error enums use it)
7. Run `cargo build --workspace --all-targets` to confirm no compile errors
8. Run `cargo clippy --workspace --all-targets -- -D warnings` to confirm clippy clean
9. Run `grep -rn "TensorParallelError" crates/ crates/dist/ crates/server/` to audit all
   call sites; ensure none use `e.to_string()` (those would have compiled either way, but
   audit for future-proofing)
</action>
