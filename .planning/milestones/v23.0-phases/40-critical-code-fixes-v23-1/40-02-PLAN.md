# Phase 40 Plan 02: Preserve Engine::step() Error Source Chain

**Goal:** Replace `Err(crate::error::EngineError::ModelError(e.to_string()))` at
`crates/core/src/engine.rs:677` with a `From`-based conversion that preserves
the underlying `vllm_traits::ModelError` as `#[source]`. Logs retain the cause
chain end-to-end (e.g., `EngineError: model failed: ModelError: candle error: ...`).

- crates/traits/src/error.rs (ModelError definition - source type)
- AGENTS.md ("Error Type Conventions" — `#[source]` chains, From impls in error/mod.rs)

<acceptance_criteria>
1. `EngineError` enum has a new variant `Model(#[source] vllm_traits::ModelError)`
   preserving the underlying source (or equivalent using `From`)
2. Existing `ModelError(String)` variant is marked `#[deprecated(since = "0.23.0",
   note = "use EngineError::Model which preserves source chain")]`
3. A `From<vllm_traits::ModelError> for EngineError` impl is added in
   `crates/core/src/error.rs` (per AGENTS.md "From<E> impls for cross-crate
   error conversion in `error/mod.rs`")
4. `crates/core/src/engine.rs:677` is updated to use `Err(crate::error::EngineError::from(e))`
   (or equivalent) — stringification is removed
5. `Display` output for `EngineError::Model(inner)` shows the nested cause chain:
   `model failed: <inner Display>`
6. All call sites that previously constructed `EngineError::ModelError(s)` are
   migrated to `EngineError::from(...)` (with one exception: if a String is truly
   needed because the source isn't a `ModelError`, a separate variant or wrapper is used)
7. `cargo build --workspace --all-targets` returns 0 errors
8. A unit test in `crates/core/src/error.rs` (or wherever EngineError tests live)
   asserts that `Display` chain shows the underlying source
</acceptance_criteria>
<action>
1. Open `crates/core/src/error.rs` and inspect the `EngineError` enum
2. Add a new variant: `#[error("model failed: {0}")]\nModel(#[source] vllm_traits::ModelError),`
   (Verify the exact import path for `vllm_traits::ModelError`; it may be `vllm_traits::error::ModelError`
   or similar — grep for current usage in the codebase)
3. Mark the existing `ModelError(String)` variant with
   `#[deprecated(since = "0.23.0", note = "use EngineError::Model which preserves source chain")]`
4. Add `impl From<vllm_traits::ModelError> for EngineError`:
   ```rust
   impl From<vllm_traits::ModelError> for EngineError {
       fn from(e: vllm_traits::ModelError) -> Self {
           Self::Model(e)
       }
   }
   ```
5. Verify `vllm_traits` is in `crates/core/Cargo.toml` `[dependencies]`
6. Open `crates/core/src/engine.rs:677` and replace:
   ```rust
   Err(crate::error::EngineError::ModelError(e.to_string()))
   ```
   with:
   ```rust
   Err(crate::error::EngineError::from(e))
   ```
7. Add a unit test (in `mod tests` block) that constructs an `EngineError::Model`
   from a fake `ModelError` and asserts the Display output contains both:
   - The outer `"model failed"` prefix
   - The inner `ModelError` Display string (e.g., a known fake message)
   Use `std::error::Error::source()` to assert the inner error is preserved.
8. Run `cargo build --workspace --all-targets` to confirm no compile errors
9. Run `cargo clippy --workspace --all-targets -- -D warnings`
10. Search for remaining uses of `EngineError::ModelError(` and migrate them or
    add `#[allow(deprecated)]` if migration is out of scope (should be only 1 site
    per the audit)
</action>
