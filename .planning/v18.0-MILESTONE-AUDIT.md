## Verdict

**gaps_found** — 3 of 14 requirements (RTE-02, RTE-03, FALL-02) are functionally unwired at the production step boundary. The building blocks are all shipped and tested, but the engine step loop does not invoke `DraftResolver` and does not handle per-sequence draft resolution. These were documented as deferred to v19.0 in the phase summaries.

**Two paths forward:**

1. **Complete milestone with deferred status** — Accept that RTE-02/03/FALL-02 are deferred to v19.0 (as documented in phase summaries). Run `/gsd-complete-milestone v18.0` to archive.

2. **Plan a closure phase** — Insert a 18.5 phase that wires the resolver into `Engine::step_speculative_inner`, adds per-sequence backend selection in batch, implements FALL-02 (catch per-seq forward errors, set `degraded_draft`, skip future drafts), and exposes v18.0 metrics via the HTTP exporter.
