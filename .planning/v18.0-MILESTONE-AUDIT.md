1. **Engine step-loop not yet wired to DraftResolver** — `step_speculative` still uses the single `draft_model` field from v17. The resolver exists and is tested, but per-request draft resolution in the actual step loop is deferred to v19. This is acceptable because the v18.0 deliverable is the registry + resolver + fallback primitives; the engine integration is a follow-up.

2. **HTTP exporter not yet exposing v18.0 metrics** — The metrics are tracked in `EnhancedMetricsCollector` but not yet exposed via Prometheus. Deferred to v19.

3. **Real-model benchmark missing** — The current bench uses stub backends. A real-model bench would require a small Llama or Qwen checkpoint. Deferred.

None of these block the milestone goals or other v19 work.
