# vLLM-lite 综合改进计划

**日期**: 2026-04-03
**目标**: 完善生产环境，提升稳定性，增加模型支持

---

## 1. Tokenizer 修复 (P0)

### 当前状态

- Placeholder 实现，返回索引而非真正 token
- 依赖 tiktoken 3.x，但 API 不稳定

### 实现方案

- 集成 HuggingFace tokenizer (tokenizers crate)
- 添加 BPE/BBPE 支持
- 确保与 Qwen3 vocab 匹配

### 验收标准

- [ ] encode/decode 正确工作
- [ ] 与 model vocab_size 匹配

---

## 2. 错误处理改进 (P0)

### 当前状态

- 多处 `.unwrap()` 调用
- 缺少错误信息上下文

### 实现方案

- 将 unwrap 改为 `?` 传播
- 添加有意义的错误信息
- 统一错误类型

### 验收标准

- [ ] 无 panic 风险
- [ ] 错误信息清晰

---

## 3. CUDA Graph 集成 (P1)

### 当前状态

- 已有 `cuda_graph.rs` 框架
- 未集成到 Engine

### 实现方案

- 在 Engine 中添加 CUDA Graph 支持
- 缓存已知形状的计算图
- 跳过相同形状的图编译

### 验收标准

- [ ] 相同 batch shape 复用 graph
- [ ] 性能提升 10%+
