- [ ] **Step 1: Read current workflow**

**Option B**: Use explicit features: `cargo clippy --all-targets --workspace --features "prometheus opentelemetry" -- -D warnings`.

Pick Option A (matches local behavior).


## Task F-3c: Remove `rustls-pemfile` (1 day, High risk)

**Files:**
- Modify: `/workspace/vllm-lite/crates/server/Cargo.toml`
- Modify: `/workspace/vllm-lite/crates/server/src/security/tls.rs` (the high-severity vuln site)

`rustls-pemfile` 2.x is unmaintained (RUSTSEC-2025-0134). No upstream fix. Migration path: use `rustls::pki_types::CertificateDer` etc. directly.

- [ ] **Step 1: Read current `tls.rs`**

```bash
cat /workspace/vllm-lite/crates/server/src/security/tls.rs
```

Identify the 3-5 places using `rustls_pemfile::{read_one, read_all, load_*}`.

- [ ] **Step 2: Replace with `rustls::pki_types`**

For each `rustls_pemfile::read_one_from_slice(&mut BufReader::new(File::open(path)?))` etc.:

```rust
// Before
use rustls_pemfile::read_one_from_slice;
let mut reader = BufReader::new(File::open(path)?);
let cert = read_one_from_slice(&mut reader)?.unwrap();

// After
use rustls::pki_types::CertificateDer;
let bytes = std::fs::read(path)?;
let cert = CertificateDer::from_pem_slice(&bytes)?;
```

(Adjust based on actual usage patterns. The exact API depends on what the code does with the parsed cert.)

- [ ] **Step 3: Remove `rustls-pemfile` dep**

```bash
# In crates/server/Cargo.toml, remove rustls-pemfile line
cargo build -p vllm-server --all-features 2>&1 | tail -10
cargo test -p vllm-server --lib security 2>&1 | tail -10  # expect TLS tests pass
```

- [ ] **Step 4: Commit**

```bash
git add crates/server/
git commit -m "fix(server): migrate from rustls-pemfile (unmaintained, RUSTSEC-2025-0134) to rustls::pki_types"
```
